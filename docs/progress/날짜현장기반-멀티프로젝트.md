# 날짜/현장 기반 일정 UI + 멀티 프로젝트 전환 기능

## 계획
- 목표: (1) 듬성듬성 떨어진 실제 날짜와 현장(캠퍼스)별로 일정을 표시하고, (2) 프로젝트 단위로 초기화/보관 후 이전 프로젝트를 다시 불러올 수 있게 한다.
- 단계:
  1. Supabase 스키마 변경 SQL 작성 (`days.site` 추가, `project_info` 다중 행 지원 + `status`, `days.project_id` 추가) → 사용자가 SQL Editor에서 직접 실행
  2. `projectStore.js`: `updateDay`, `listProjects`, `startNewProject`, `loadProject` 액션 추가 및 `fetchFromDB`/`addDay`/`initDB`를 활성 프로젝트 기준으로 수정
  3. `DayNavigation.jsx`: 인덱스 계산 대신 실제 `day.date` 표시, 탭에 현장명 표기로 시각적 그룹핑
  4. `DayModule.jsx` 또는 별도 컴포넌트: 관리자 모드에서 일자별 날짜/현장명 편집 UI 추가
  5. `Dashboard.jsx` 또는 `TopBar.jsx`: 관리자 모드 전용 "새 프로젝트 시작" / "이전 프로젝트 불러오기" UI 추가
  6. 로컬 dev 서버(`npm run dev`)로 동작 확인 (배포는 보류)
- 상태: 완료

## 완료 내역
- 한 일:
  - Supabase 스키마 확장 (`docs/progress/schema-migration.sql`): `days.site`, `days.project_id`, `project_info.status`/`created_at` 추가, `project_info.id`를 text로 변경. 기존 데이터는 `project_id='1'`로 연결.
  - `projectStore.js`에 `activeProjectId` 상태와 `updateDay`, `listProjects`, `startNewProject`, `loadProject`, `deleteProject` 액션 추가. `fetchFromDB`/`addDay`/`initDB`가 `status='active'` 프로젝트 기준으로 동작하도록 수정.
  - `DayNavigation.jsx`: 순번 기반 라벨을 실제 `day.date` 기반으로 변경, 탭에 현장명(`day.site`)을 함께 표시해 시각적으로 그룹핑.
  - `DayModule.jsx`: 관리자 모드에서 일자별 날짜/현장명을 직접 입력하는 UI 추가. 작업자(worker) 모드에서는 "진행 상태" 버튼을 비활성화해 관리자만 상태 변경 가능하도록 제한.
  - `Dashboard.jsx`: "프로젝트 관리" 카드 추가 — 새 프로젝트 시작(기존 프로젝트 자동 보관), 이전 프로젝트 목록 조회/불러오기, 보관된 프로젝트 삭제(활성 프로젝트는 삭제 버튼 숨김) 기능 구현.
  - `TopBar.jsx`: 로고 이미지 교체 및 가운데 정렬/크기 조정 (별도 요청 사항, 같은 세션에서 함께 처리).
- 변경 파일: `src/store/projectStore.js`, `src/components/DayNavigation.jsx`, `src/components/DayModule.jsx`, `src/components/Dashboard.jsx`, `src/components/TopBar.jsx`, `docs/progress/schema-migration.sql`
- 확인 방법: `npm install`(외장 USB 드라이브 쓰기 캐싱 정책을 "장치에 쓰기 캐시 사용"으로 확인 후) → Supabase SQL Editor에서 마이그레이션 실행 → `npm run dev`로 로컬(`localhost:5173`) 구동 후 관리자/작업자 모드 전환하며 기능 확인.
- 상태: 완료
