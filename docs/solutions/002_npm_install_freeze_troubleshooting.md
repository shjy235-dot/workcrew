# NPM/PNPM 의존성 설치 멈춤(Freeze) 현상 대책 및 향후 방안

## 1. 문제 원인 분석
Windows 환경에서 `npm install` 또는 `pnpm install` 실행 시 패키지 추출(압축 해제) 단계에서 무한정 대기(Freeze)하는 현상의 주요 원인은 다음과 같습니다:
- **Windows Defender(백신) 실시간 감시**: `node_modules`에 수만 개의 작은 파일들이 생성/수정/링크되는 과정을 백신이 하나하나 스캔하면서 디스크 I/O가 극심하게 병목을 일으킴.
- **포스트 인스톨(post-install) 스크립트 행(Hang)**: 특정 패키지의 설치 후 빌드 스크립트가 C++ 빌드 도구를 찾지 못하거나 네트워크로 바이너리를 받아오려다 실패하면서 프로세스가 멈춤.
- **네트워크 타임아웃 및 레지스트리 병목**: IPv6 이슈 혹은 npm 공식 레지스트리의 일시적 지연.

## 2. 전반적인 대책 (즉시 적용)
1. **의존성 무결성 캐시 정리**: 꼬여있는 캐시를 먼저 비운다. (`npm cache clean --force`)
2. **무거운 스크립트 실행 방지**: `--ignore-scripts` 옵션을 통해 빌드 시 행이 걸릴 수 있는 스크립트를 건너뛴다.
3. **불필요한 감사/펀딩 로그 생략**: `--no-audit --no-fund`를 통해 설치 후 처리 시간을 단축한다.
4. **Peer Dependency 무시**: 리액트 생태계에서 잦은 버전 충돌로 인한 롤백/재탐색을 막기 위해 `--legacy-peer-deps`를 사용한다.

## 3. 향후 오류 발생 시 에이전트(AI)의 대응 방안
향후 모든 프로젝트에서 에이전트가 패키지 설치 시 아래 프로세스를 자동으로 준수하도록 규칙을 제정합니다.

1. **설치 명령 표준화**: 특수한 요구사항이 없다면 항상 `npm install --no-audit --no-fund --legacy-peer-deps`를 기본으로 사용한다.
2. **타임아웃 및 강제 우회 모니터링**: 
   - 설치 개시 후 2~3분 이상 진척이 없다면 프로세스를 즉시 강제 종료(`kill`)한다.
   - 즉시 `node_modules`를 삭제하고 `npm install --ignore-scripts --no-audit --no-fund --legacy-peer-deps`로 우회하여 재설치한다.
3. **Windows Defender 예외 처리 권고**: 빈번하게 발생할 경우 사용자에게 `node_modules` 폴더를 백신 실시간 감시 예외로 등록하도록 안내한다.
4. **터미널 우회(Bypass)**: 권한 문제로 인해 설정이 막히는 경우 파워쉘 강제 명령어(예: `Add-Content`)를 통해 전역 설정(`AGENTS.md`)에 규칙을 스스로 즉시 주입한다.
